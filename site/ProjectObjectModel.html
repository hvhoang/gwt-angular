"Project Object Model" is a fancy name for a Maven project descriptor. Most people refer to this as "pom," and its filename is pom.xml. Among many other things, it declares the project's depenency. For this tutorial project, we have two notable dependencies.

com.asayama.gwt:gwt-angular-ng project refers to the core API for GWT Angular. It provides the GWT wrapper around AngularJS's "ng" module.

com.asayama.gwt:gwt-angular-ngRoute provides the GWT wrapper around AngularJS's "ngRoute" module. 

<dependencies>
  <dependency>
    <groupId>com.asayama.gwt</groupId>
    <artifactId>gwt-angular-ng</artifactId>
    <version>{{ GWT_ANGULAR_VERSION }}</version>
    <scope>provided</scope>
  </dependency>
  <dependency>
    <groupId>com.asayama.gwt</groupId>
    <artifactId>gwt-angular-ngRoute</artifactId>
    <version>{{ GWT_ANGULAR_VERSION }}</version>
    <scope>provided</scope>
  </dependency>
</dependencies>

In order to compile Java code into JavaScript, we must invoke GWT compiler at the time of the build. We can do this by adding the plugin, gwt-maven-plugin to our POM.

In this example, we specify com.asayama.gwt.angular.tutorial.SampleEntryPoint as the compile target GWT module. This module declaration references the file src/main/java/com/asayama/gwt/angular/tutorial/SampleEntryPoint.gwt.xml file in the project. 

<plugin>
  <groupId>org.codehaus.mojo</groupId>
  <artifactId>gwt-maven-plugin</artifactId>
  <version>{{ GWT_VERSION }}</version>
  <executions>
    <execution>
      <configuration>
        <module>com.asayama.gwt.angular.tutorial.SampleEntryPoint</module>
      </configuration>
      <goals>
        <goal>compile</goal>
      </goals>
    </execution>
  </executions>
</plugin>
